Version control

  • Keeps track of the changes to your code
  • Allows you to go back to any point in history
  • Allows you to bundle code changes together into commits
  • Allows you to merge history

Git is a version control system

Git global setup

git config --global user.email "you@example.com"
git config --global user.name "Your Name"

# If Windows
git config --global core.safecrlf false

Setup a project directory

mkdir bugfree-unicorn
cd bugfree-unicorn

Initialize a repository

git init

Check the status of the repository

git status

Create a README.md file

# Windows
C:/Users/<user>/bugfree-unicorn
# Mac/Linux
/users/<user>/bugfree-unicorn
# or with a command
touch README.md
echo "## bugfree-unicorn" > README.md

Tell git to track the README.md file

git add README.md

Bundle this change into a commit

git commit -m 'Add project name to README'

Create project on Github

Git remotes

git remote add origin <github-url>

Push your pending commit(s) to remote

git push

Git team workflow

  • Working in separate branches of history
  • Create a branch
  • Make commits on the new branch
  • Merge the changes back to the master branch

Create a branch

git branch add-list
git checkout add-list

Make some changes and commit them

# Add a list item to your README.md
- Hi I am a list item
- I am another list item
git commit -am 'Add list items'

Let's look at our commit history

git log

Oh no! There's a bug on master

git checkout master

# Change `##` to `#`

git commit -am 'Fix header'

# Go back to working branch

git checkout add-list

Let's squash some of those commits

git rebase -i <sha>

Merge your branch into master

git checkout master

git merge add-list

Delete the old branch

git branch -d all-list
